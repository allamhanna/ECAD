- [x] **M7** — Auto-connect wiring: orthogonal routing, junctions, auto wire numbering (same-page only — cross-page "interruption points" deferred to a focused follow-up)
  - [x] Pin geometry finally used for real: `SymbolConnectionPoint.X/Y/Direction` (parsed since M4, unused until now) transformed by each placement's position/rotation/mirror (`PlacementPinGeometry`) to get every pin's live world position and outward-facing direction. Pins render as small markers on every placement.
  - [x] Manual wire drawing: click-drag from one pin to another draws a live dashed preview and creates a `Connection` on release. Wires have no stored route — `OrthogonalRouter` recomputes a straight-or-one-bend path fresh from both pins' current positions on every render, so a wire visibly re-routes as either endpoint's placement moves, with no extra state to keep in sync.
  - [x] Auto-connect and auto-disconnect, direction-aware (redesigned twice from user feedback during testing): two pins connect only when they sit on the same grid row/column *and* their connection-point directions point at each other (e.g. a pin exiting right only joins another pin further along that row exiting left back toward it) — not merely touching, and not merely sharing a line. This check re-runs after every place/attach/move/rotate: sliding a connected part further along its line keeps the wire (just longer), moving it off that line drops the connection outright, matching a real user expectation that a wire shouldn't just elastically follow a part wherever it goes. A pin landing mid-span on an existing wire's path still auto-joins it too (a T-junction). While a part is being dragged, its wires are hidden entirely rather than stretching live to the cursor, since whether it ends up connected is only decided on drop.
  - [x] Magnetic snap while placing/dragging: pulls the relevant axis (row for a left/right-facing pin, column for up/down) into exact alignment with a nearby *compatible* (opposite-facing) pin — without this, landing the required exact grid alignment by eye is impractical since pin offsets aren't visible before placing.
  - [x] Junctions: a filled dot renders wherever 3+ connections share an endpoint, or one connection's endpoint lands inside another's routed path — computed purely from the existing pairwise `Connection` records (`JunctionDetector`), no separate junction entity needed.
  - [x] Auto wire numbering: sequential, project-wide (one fixed scope, not Section 6.1's "configurable per-page-or-project" toggle — same simplification precedent as ADR-008's Designation suggestion), assigned automatically on every connection creation. Double-click a wire to rename its number (uniqueness-validated); a "Renumber Wires" toolbar button reassigns every connection in the project in page order.
  - [x] Wire select/delete/undo, same shape as placement select/delete from M5.
  - [x] Necessary fix found while building this: `ProjectSession.DeletePlacement` now deletes any `Connection`s touching the placement's pins *before* deleting the pins themselves — `Connection`'s FKs to `DevicePin` have no `ON DELETE CASCADE`, so deleting a wired placement would otherwise throw. Undoing the delete restores the placement but not its former wires (same recreate-not-restore acceptance as ADR-007).
  - [x] `ProjectSession.ConnectionsChanged` event added, exact same shape as M6's `PlacementsChanged`, so wires stay live across every open page window, not just the one where a change happened.
